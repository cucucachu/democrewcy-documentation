Development Process

Rules

  • Don't fuck up master!
  • Always work in branches, no direct pushes to master.
  • Leave comments on you functions.
  • Try to keep these docs up to date.
  • Tests please!

Process

  1. Find or create an issue to work on. Assign yourself to it.
  2. Update the Trello board so we can see what you're working on.
  3. Create a fresh branch from master. Use format "issue-" followed by the issue number. i.e. "issue-1"
  4. Do your work on your branch.
  5. Commit often, and leave helpful commit messages.
  6. Add tests for your changes, and update existing tests as necessary.
  7. Add comments to your code and update documentation as necessary.
  8. Once ALL tests are passing, you can merge back to master.
  9. Merge back to master.
  10. Close github issue with comment explaining what you added, what you changed, and your testing.
  11. Update Trello board.
